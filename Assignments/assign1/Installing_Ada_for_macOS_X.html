Installing Ada on Mac OS X (Mountain Lion or later)

To install Ada on your Mac, perform the following steps:

(1) In order to use GNU Ada (GNAT) on your Mac, you will need to have certain GNU utilities installed, such as "make" and "gcc".  If these are already on your computer, then you can skip to step (2).

Although you get make, gcc, etc. as part of XCode, which requires a 3.5GB  download, it is sufficient just to download the XCode command line tools (which is a much smaller download).  To do this, in a shell (e.g. the terminal application), type:

xcode-select --install

A window will pop up asking if you would like to install the command line tools. Click "Install" and the command line tools will be installed.

(2) Go to http://libre.adacore.com/download/configurations and, in the middle of the page, where it says "Select your platform", choose "x86-64 Mac OS X (64 bits)" and "GNAT GPL 2017".

(3) Expand the portion labeled "GNAT Ada 2017" and click on the link that says "gnat-gpl-2017-x86_64-darwin-bin.tar.gz" to download and save the file on your computer. Then, uncompress the downloaded file.

(4) In a shell (e.g. terminal or under emacs), navigate down into the resulting directory until you see the files README, doinstall, etc.  Then, type,

sudo ./doinstall

You will be asked for the administrator password (which is likely just your password on your Mac).  You should accept the default directory in which the files are to installed.

(5)  Once the installation has completed (and the files are in /usr/local/gnat), you will need to add "/usr/local/gnat/bin" to the front of the PATH environment variable.  To do this, you can put one of the following lines in your shell configuration files (e.g. .bashrc, .cshrc, etc.)

      PATH=/usr/local/gnat/bin:$PATH; export PATH  (Bourne shell)
      setenv PATH /usr/local/gnat/bin:$PATH        (C shell)

 

Testing the compiler

In the Ada resources folder, click on "Simple Ada program to test compiler".  A zip file will be downloaded.  When you unzip it, you will see several files with .ads and .adb extensions.  In a shell, go to the directory containing the files. Then type,

gnatmake  objmain.adb

You will see that gcc is invoked several times, along with gnatbind and gnatlink.

To run the program, type

./objmain

(you won't need the "./" if your PATH environment variable contains ".")
